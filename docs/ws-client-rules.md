# WS 测试页面规则

适用文件：

```text
examples/ws-client.html
examples/frame-sync.html
```

## 基础约束

- 示例页面规则只维护在本文档中，不为每个页面新增独立规则文档。
- 使用单文件 HTML。
- 使用 Vue CDN。
- 不引入构建工具。
- 不引入 npm 依赖。
- CSS、HTML、JavaScript 保持内嵌。
- 页面可直接用浏览器打开。
- 默认 WebSocket 地址为 `ws://localhost:10100/ws`。

## 通用协议

请求包：

```json
{
  "cmd": 1001,
  "subCmd": 1,
  "data": {}
}
```

响应包：

```json
{
  "cmd": 1001,
  "subCmd": 1,
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

## 通用测试页

`examples/ws-client.html` 应支持：

- 连接 WebSocket。
- 断开 WebSocket。
- 显示连接状态。
- 选择发包预设。
- 编辑 `cmd`。
- 编辑 `subCmd`。
- 编辑 `data JSON`。
- 格式化 `data JSON`。
- 发送协议包。
- 显示发送、接收、错误、关闭日志。
- 清空日志。

新增模块或命令时，需要同步更新 `examples/ws-client.html` 的 `presets`。

## 帧同步测试页

`examples/frame-sync.html` 应支持：

- 输入 ID。
- 确认 ID。
- 确认 ID 时必须先发送检查请求。
- 如果该 ID 已存在同步房间，则重连房间并从第 1 帧开始追帧。
- 如果该 ID 不存在同步房间，才允许加入匹配或退出匹配。
- 两个客户端匹配成功后开始同步。
- 同步开始后定时提交输入快照。
- 接收服务端推送帧并更新最新帧号。
- 主要同步展示区域使用 canvas。
- canvas 中至少显示一个可通过上下左右输入移动的方块。
- 方块位置必须在收到服务端帧后更新，而不是只做本地即时移动。
- 重连追帧时必须显示追帧进度。
- 追帧推荐使用 2 倍速度，当前页面以 30ms 间隔播放历史帧。
- 桌面端布局应尽量控制在首屏内，避免出现页面级滚动条。

帧同步页面不要显示“游戏”等字样，使用“帧同步”“同步”“匹配”“房间”“画布”“客户端”等表述。

## 帧同步命令

| Action | cmd | subCmd |
| --- | ---: | ---: |
| 检查已有房间 | 5001 | 1 |
| 加入匹配 | 5001 | 2 |
| 退出匹配 | 5001 | 3 |
| 提交输入快照 | 5001 | 4 |
| 服务端推送 | 5001 | 6 |

## 维护规则

修改示例页面前，先读取本文档。

当新增 Go 模块或命令时，需要同步更新：

- `examples/ws-client.html` 的 `presets`。
- 本文档的相关命令或交互规则。
- README 中必要的示例说明。

当新增或修改示例页面时：

- 只更新本文档，不新增页面专属规则文档。
- 如果示例页面有特殊交互规则，将规则追加到本文档对应章节。
- 如果示例页面对应新命令，将命令追加到本文档相关协议说明中。
